# 📝 Экзамен по JavaScript

> **Дисциплина:** Проектирование пользовательских интерфейсов  
> **Тема:** Основы JavaScript — условия, циклы, массивы  
> **Время выполнения:** 90 минут

---

## 🚀 Начало работы

1. **Примите задание** по ссылке в GitHub Classroom.
2. **Клонируйте** свой репозиторий:
   ```bash
   git clone <ваша-ссылка>
   cd <имя-репозитория>
   ```
3. **Выполните задания** (см. ниже).
4. **Проверьте свой код одним из способов ниже.**
5. **Сохраните результат:**
   ```bash
   git add .
   git commit -m "done"
   git push
   ```
   После push **автоматически запустится проверка на GitHub** и будет выставлен балл.

## 📂 Структура репозитория

```
tasks/
  task01/   ← Чётные числа
  task02/   ← Сумма массива
  task03/   ← FizzBuzz
  task04/   ← Максимальный элемент
  task05/   ← Переворот строки
  task06/   ← Подсчёт вхождений
  task07/   ← Сглаживание массива
  task08/   ← Калькулятор оценок
  task09/   ← Суммы строк матрицы
  task10/   ← Уникальные элементы
```

Каждая папка содержит два файла:

- **`task.js`** — здесь нужно написать код (там, где написано `// Ваш код здесь`)
- **`task.test.js`** — тесты (не изменять!)

## 🧪 Способ проверки: Локальное тестирование

У вас есть **два варианта** проверить свой код локально:

### ✅ Способ 1: Через браузер (БЕЗ Node.js)

Если у вас **не установлен Node.js**, используйте браузерный интерфейс:

1. Откройте файл **`test-browser.html`** в браузере (двойной клик)
2. Нажмите кнопку **"▶️ Запустить тесты"**
3. Увидите результаты всех 10 заданий

![Браузерный тестер](https://via.placeholder.com/600x400?text=Browser+Test+Interface)

---

### ✅ Способ 2: Через Node.js и терминал (если установлен)

Если у вас **установлен Node.js**, используйте командную строку:

1. **Установите зависимости:**

   ```bash
   npm install
   ```

2. **Запустите все тесты:**

   ```bash
   npm test
   ```

3. **Запустите тесты одного задания:**
   ```bash
   npx jest tasks/task01 --verbose
   ```

Результаты также будут выведены в консоль.

---

### 🤖 Автоматическая проверка на GitHub

После каждого `git push`:

1. GitHub Actions **автоматически** запускает все тесты
2. Результаты видны во вкладке **Actions** вашего репозитория
3. Балл выставляется в журнал GitHub Classroom

**Не нужно ничего делать вручную!** Тесты запустятся сами.

## 💡 Советы

- Читайте комментарии в файле `task.js` — там описаны условие и примеры.
- Функция уже объявлена, вам нужно только написать тело.
- **Не изменяйте** `task.test.js` и строку `module.exports` в `task.js`.
- **Способ 1 (браузер):** Подходит для быстрой проверки без установки Node.js.
- **Способ 2 (Node.js):** Дает подробные отчеты об ошибках.
- Тестируйте часто — исправляйте ошибки сразу же.
- Если тест не проходит — внимательно читайте сообщение об ошибке.
