# ⚙️ Как работает автоматическая проверка на GitHub

## 📍 Что происходит при `git push`?

1. **GitHub Actions автоматически запускается**
   - Прочитайте конфиг: `.github/workflows/classroom.yml`

2. **Система устанавливает Node.js 20**
   - Закачивает все зависимости через npm

3. **Запускаются тесты**
   - Jest проверяет каждое из 10 заданий
   - Результаты сохраняются в `results.json`

4. **Вычисляется итоговый балл**
   - Формула: (пройденные_задания / 10) × 100

5. **Отправляется отчет в GitHub Classroom**
   - Вы видите оценку в журнале

## 📊 Результаты где?

### Во вкладке "Actions"
```
GitHubRepo > Actions > Автоматическое задание > Нажмите на любой push
```

Там вы увидите:
- ✅ Какие задания прошли
- ❌ Какие задания не прошли
- 📈 Итоговый балл

### Пример вывода:
```
=== РЕЗУЛЬТАТЫ ЭКЗАМЕНА ===
✅ Задание 01: 5/5 тестов
✅ Задание 02: 5/5 тестов
❌ Задание 03: 0/5 тестов
...
Пройдено заданий: 9/10
Пройдено тестов:  45/50

ИТОГОВЫЙ БАЛЛ: 90/100
```

## ⚡ Как тесты работают?

Каждое задание имеет **5 тестов**, которые проверяют:
- Базовые случаи
- Граничные случаи (пустые массивы, нули и т.д.)
- Специальные условия

Задание считается **выполненным**, если **все 5 тестов прошли**.

## 🔧 Может ли что-то пойти не так?

### Тесты не запускаются
- Проверьте, что `npm install` пройдет успешно
- Убедитесь, что jest установлен в `package.json`

### Мой код работает локально, но тест не проходит на GitHub
- Может быть проблема с форматированием
- Проверьте, что не изменили `module.exports`
- Убедитесь, что функция возвращает правильный тип

### Как пересдать?
- Исправьте код
- Сделайте `git commit` и `git push`
- GitHub Actions запустится автоматически еще раз

## 📝 Файлы, которые не должны изменяться

❌ **НЕ ИЗМЕНЯЙТЕ:**
- `task.test.js` в каждой папке
- Строку `module.exports` в `task.js`
- Имена функций

✅ **ИЗМЕНЯЙТЕ ТОЛЬКО:**
- Тело функции (там, где написано `// Ваш код здесь`)
